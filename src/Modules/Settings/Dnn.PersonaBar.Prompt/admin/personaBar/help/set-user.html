set-user

Enables you to set various values related to the specified user.

Options

Flag Value Required Default Description
--id integer Yes The UserId for the user being updated. This value is required, but if you pass the value as the first argument after the command name, you do not need to explicitly use the --id flag. (see the examples below)
--email string No The email address to set for the user.
--username string No The username to set for the user.
--displayname string No The name to be displayed for the user.
--firstname string No The first name of the user.
--lastname string No The last name of the user.
--approved boolean No Approval status of the user.
--password string No The new password to assign to the user. The password must be valid according to the site's password rules.

Approve/UnApprove a User

Approves/UnApproves the user with a User ID of 345. In this example, true is passed for the --approve flag.

set-user 345 --approve true

Change a User's Username

Changes the username of the user with a User ID of 345 to john.smith.

set-user 345 --username "john.smith"

Change a User's Name Properties

Sets the first, last, and display name for the user with an ID of 345. Note that John and Smith do not need to be surrounded by quotes but Johnny Smith does since it is more than one word.

set-user 345 --firstname John --lastname Smith --displayname "Johnny Smith"
